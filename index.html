How it works?

1. Understand intent The agent first classifies whether your message is a navigation request, a study-space planning question, or general chat, and chooses the appropriate tools and tone.
2. Use tools only when needed Navigation queries trigger Google Routes calls for live travel times and route geometry; study planning uses the local candidate dataset and the LLM for natural recommendations.
3. Route parsing and start-point fallback The agent will automatically parse origin and destination from free text (e.g. "from South Kensington to White City"). If no explicit origin is found, it uses the start point selected on the left (map selection or campus preset).
4. Live data and verification When available, Google provides live duration and distance estimates and encoded polylines for map preview. The assistant reports exact numbers from the tool and links to Google Maps for full directions.
